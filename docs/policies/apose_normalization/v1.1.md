# A-Pose Normalization v1.1

## 1. Summary
A-Pose Normalization v1.1은 모든 신체 치수 측정의 전제 조건으로 사용되는
기준 포즈 정규화 정책이다.

입력 SMPL-X 바디를 측정 및 의류 피팅에 적합한 표준 A-Pose로 변환하며,
좌표계, dtype, 단위를 강제하여 downstream 측정의 재현성과 안정성을 확보한다.

---

## 2. Goal

### Primary Objective
- 모든 신체 측정 파이프라인에서 **일관된 기준 포즈(A-Pose)** 제공
- 팔/겨드랑이 간섭으로 인한 측정 노이즈 제거

### Constraints
- 연산 비용은 전체 파이프라인 대비 미미해야 함
- downstream 측정 결과에 부정적 편향을 유발하지 않아야 함
- 입력 데이터 분포 변화에도 안정적으로 동작해야 함

---

## 3. Scope

### In Scope
- Phase 2 (Smart Mapper)
- Offline / Training
- 모든 신체 측정 정책의 upstream dependency

### Out of Scope
- 포즈 추정 자체의 정확도 개선
- 사용자 입력 오류 보정
- 의상 시뮬레이션 로직

---

## 4. Invariants / Rules
> 본 정책이 **항상 강제하는 불변 규칙**

- **Coordinate System**
  - SMPL-X 기준 좌표계 유지
- **Axis Convention**
  - 좌우/상하/전후 축 정의를 변경하지 않음
- **dtype / device**
  - float32 고정
  - 명시적으로 지정된 device만 사용
- **Units / Scale**
  - 모든 길이 단위는 meter 기준
- **Determinism**
  - 동일 입력에 대해 동일 출력 보장

---

## 5. Gate Definitions (Verification Gates)
> 본 정책이 유효하다고 판단되기 위해 **반드시 통과해야 하는 검증 게이트 정의**

### Gate G1: A-Pose Angle Range Stability
- **Intent**
  - A-Pose 변환 결과가 해부학적으로 안정적인 각도 범위를 유지하는지 검증
- **Check**
  - 어깨, 팔꿈치, 팔 각도가 사전에 정의된 허용 범위 내에 있는지 확인
- **Pass Criteria**
  - 모든 샘플에서 각도가 허용 범위를 벗어나지 않음
  - 극단 포즈 입력에서도 팔/겨드랑이 간섭 없음
- **Fail Criteria**
  - 허용 범위를 초과하는 각도 발생
  - self-collision 또는 명백한 포즈 왜곡 발생

---

### Gate G2: Axis Contract Enforcement
- **Intent**
  - 포즈 정규화 과정에서 좌표계 및 축 규약이 깨지지 않았는지 검증
- **Check**
  - 출력 포즈의 좌우/상하/전후 축 방향 일관성 확인
- **Pass Criteria**
  - 모든 출력이 정의된 axis contract를 정확히 준수
  - 좌우 반전 또는 축 혼동 사례 없음
- **Fail Criteria**
  - axis 방향 불일치
  - 좌우 반전, 축 swap 발생

---

### Gate G3: dtype / device Enforcement
- **Intent**
  - 연산 안정성과 재현성을 위해 dtype 및 device 규약이 강제되는지 검증
- **Check**
  - 모든 중간/최종 텐서의 dtype 및 device 확인
- **Pass Criteria**
  - float32 유지
  - 명시된 device 외 사용 없음
- **Fail Criteria**
  - 암묵적 캐스팅 발생
  - device mismatch 또는 혼용

---

### Gate G4: Downstream Measurement Stability
- **Intent**
  - A-Pose 정규화가 downstream 신체 측정의 안정성을 실제로 개선하는지 검증
- **Check**
  - 동일 입력에 대해 Shoulder, Girth 등 주요 측정값 변동성 비교
- **Pass Criteria**
  - 이전 버전 대비 변동성 감소 또는 동일 수준 유지
  - 측정 결과에 구조적 왜곡 없음
- **Fail Criteria**
  - 변동성 증가
  - 특정 체형/포즈에서 측정 실패 또는 이상치 발생

---

## 6. Acceptance Criteria (Freeze Decision)
> 본 정책이 **Candidate → Frozen** 으로 승격되기 위한 조건

- Gate G1 ~ G4가 모두 **PASS**
- 동일 입력에 대한 재현성 확인
- downstream 정책에 부정적 영향 없음
- 본 정책을 검증하는 Report가 최소 1개 이상 존재할 것
