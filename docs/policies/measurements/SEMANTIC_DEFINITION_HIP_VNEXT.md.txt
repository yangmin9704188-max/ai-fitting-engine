본 문서는 HIP이 무엇을 의미하는지를 정의한다.
이후 Layer는 이 정의를 전제로만 동작한다.

1. Core Semantic Statement

HIP은 인체 하부 몸통(lower torso) 영역에서,
골반(pelvic structure)과 둔부(gluteal mass)를 포함하는
의미적으로 안정적인 둘레 측정이다.

HIP은 단일 기하학적 극값이 아니라,
의복 핏과 체형 분류에 의미가 있는 골반 둘레를 의미한다.

2. Distinction from WAIST / THIGH

WAIST: 복부 중심, 가장 좁은 둘레 (upper abdomen)

HIP: 골반 및 둔부 포함, 하부 몸통의 최대 둘레

THIGH: 사지(leg) 영역, 골반과 분리

HIP은 “허리 아래의 최대값”이 아니라,
골반을 중심으로 한 하부 몸통의 대표 둘레다.

이 구분은:

WAIST의 기하학적 최소 규칙과 혼동을 방지하고

THIGH로의 경계 이탈을 막기 위함이다.

3. Defined Region (의미론적 영역)

HIP은 다음 의미론적 영역에서만 정의된다.

Lower Torso (하부 몸통) 내

골반 중심부를 포함

대퇴(THIGH) 영역은 제외

정확한 좌표 비율, landmark, skeleton 기반 정의는
Semantic Layer에서 의도적으로 고정하지 않는다.

4. Selection Semantics (의미 수준)

HIP은 하부 몸통 영역에서
골반을 대표하는 최대 둘레로 정의된다.

중요:

“최대”는 기하학적 max 연산을 강제하지 않는다

이는 의미적 설명이며, 구현 선택은 Geometric Layer의 책임

5. Undefined Semantics (NaN = Undefined)

HIP은 다음 경우 정의되지 않음(Undefined) 으로 간주한다.

하부 몸통 영역에서 폐곡선 단면이 형성되지 않는 경우

골반 구조를 포함하는 단면 후보가 존재하지 않는 경우

입력 형상이 하부 몸통을 충분히 표현하지 못하는 경우

NaN은 실패가 아니라, HIP이 성립하지 않음을 의미한다.

6. Unit Assumption

HIP의 의미론은 입력 좌표가 meters 단위임을 전제로 한다.

단위 위반 시 결과는 의미론적으로 해석 불가

보정·추론 책임은 Semantic/Contract Layer에 귀속

7. Semantic Boundary Statement

본 Semantic Definition은 다음을 의도적으로 포함하지 않는다.

단면 생성 방식

후보 선택 규칙

골반 landmark 추론

검증 기준 및 임계값

Semantic Layer는 오직 다음 질문에만 답한다.

“HIP이란 무엇을 측정한다고 말하는가?”
