AI Fitting Engine
Lightweight R&D & Development Guide (Single-Developer)
1. 프로젝트 개요

이 프로젝트는 1인 개발 환경에서 AI 기반 가상 피팅 엔진을 연구·구현·검증하기 위한 장기 R&D 프로젝트이다.

핵심 목표는 다음과 같다.

신체 측정 및 가상 피팅을 위한 안정적이고 재현 가능한 AI 파이프라인 구축

과도한 거버넌스를 제거하고, 문서–코드–실험 간 정합성을 유지하는 경량 구조 확립

AI 에이전트 분업을 활용하여 1인 개발의 한계를 구조로 보완

본 README는 “무엇을 어떻게 진행하는 프로젝트인가”를 한눈에 파악하기 위한 기준 문서이며,
세부 규칙과 절차는 각 가이드 문서에 위임한다.

2. 전체 개발 흐름 (핵심 개념)

이 프로젝트의 연구·개발 흐름은 아래의 고정된 개념 체계를 따른다.

Policy  →  Spec  →  Run  →  Report
(기준)     (구현 설명)  (실행 기록)  (판정/승인)


각 문서/산출물의 역할은 명확히 분리된다.

Policy: 무엇이 올바른가 (Gate 정의, 기준)

Spec: 어떻게 구현되어야 하는가 (설계 또는 역설계)

Run: 실제로 무엇을 실행했는가 (사실 기록)

Report: 그래서 기준을 통과했는가 (판단)

이 분리는 프로젝트 전반에서 가장 중요한 불변 원칙이다.

3. 폴더 구조 (공식)
3.1 제품 및 실행 로직

engine/

향후 제품 로직의 중심이 될 디렉토리

현재는 비어 있으며, 점진적으로 이전 예정

core/

재사용 가능한 순수 로직

측정 함수, 필터, 수학적 유틸리티 등

pipelines/

실행 파이프라인

실험 및 배치 처리 흐름

verification/

검증 러너 및 검증 도구

tests/

테스트 스위트

3.2 실행 기록 (Run 정본)

runs/

실행 단위 기록의 정본

Run 1개 = 디렉토리 1개

최소 run.md 필수

판단(PASS/FAIL)은 포함하지 않음

Run은 미래 실행부터 기록하며, 과거 실행을 억지로 복구하지 않는다.

3.3 문서

docs/

policies/ : 정책 문서 (Gate 정의)

specs/ : 구현 명세 (설계 / 역설계)

reports/ : 검증 결과 및 승인 판단

sync/ : Sync Hub 관련 문서

archive/ : 레거시 문서 (참고용, 강제 규칙 아님)

문서의 정본(Source of Truth)은 Git에 저장된 MD 파일이다.

3.4 데이터 및 모델

data/

처리된 데이터

models/

SMPL-X 및 관련 모델 파일

3.5 운영 자동화 도구

tools/

db_upsert.py : 메타데이터 DB 업서트 단일 진입점

sync_state.py : Sync Hub 상태 갱신

기타 운영 보조 스크립트

3.6 데이터베이스

db/metadata.db

정책 / 리포트 메타데이터 인덱스

검색 및 요약 목적

문서 정본은 저장하지 않음

DB는 보조 인덱스이며, 프로젝트의 중심은 문서와 코드이다.

6. 이 README의 역할

이 문서는 다음을 위한 기준점(anchor) 이다.

개발 순서와 구조를 잃지 않기 위해

AI에게 프로젝트 맥락을 전달하기 위해

“지금 이 프로젝트가 어디쯤 와 있는가”를 빠르게 파악하기 위해

세부 규칙은 이 문서를 넘어서지 않는다.

7. 핵심 원칙 요약

문서가 정본이다

Run은 사실만 기록한다

판단은 Report에서만 한다

구조는 단순하게 유지한다

과거를 미화하지 않고, 현재부터 정리한다

끝으로

이 README는 완성 문서가 아니라, 방향을 잃지 않기 위한 기준 문서다.
프로젝트가 성장해도, 이 문서의 역할은 바뀌지 않는다.
